# Kaffeemaschine

10. Februar 2026: komplette Überarbeitung und konsequentere Splittung anhand der OOP-Paradigmen.
Die main()-Methode befindet sich in CoffeeMachine.java
- Menüs sind in separate Klassen aufgeteilt
- Über Enums werden die Zustände gesteuert
- Spinner für die Brüh-Simulation ist in eine separate Klasse ausgelagert
- Menü, Maschine, Behälter, Kaffee-Rezepte etc. sind sauber voneinander getrennt
- Klassen folgen dem Single Responsibility Principle
- Lose Kopplung wurde größtenteils eingehalten - stetige Verbesserungen folgen

20. August 2025: Ersten Entwurf der Java-Dateien abgelegt. Diese dienen als Basis für die weitere Entwicklung.

Inhalt:
Dieses Repository dient Übungszwecken in Java. Ich erstelle eine Kaffeemaschine, die sich an OOP-Paradigmen hält,
die ein robustes Fehlerhandling für Nutzereingaben enthält und das UI von der Geschäftslogik und (evtl.) der Datenhaltung trennt.
